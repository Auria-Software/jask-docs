# Les actions Internes

Ces actions s’exécutent en arrière-plan pour automatiser et enrichir le scénario. 

## Les fonctionnalités

### Déléguer à un assistant

Mobilisez un agent de votre catalogue, spécialisé et connecté à une base de données pour traiter une partie du scénario. Le texte de la brique précédente sera passé en contexte du prompt à l'agent.

![experience_internal_action](@public/experiences/agent.png)

### Exécuter une application

Déclenchez une application déjà existante dans JAsk. Le résultat de l'application affiché et passé à la prochaine brique est le dernier résultat de l'application.

<Callout type="warning">
Assurez-vous que l’entrée que vous mettez à cette brique, correspond bien au contexte nécessaire au lancement de l’application (est-ce du texte ? ou des documents ?).
</Callout>

![experience_internal_action](@public/experiences/app.png)


### Combiner des résultats

Fusionnez plusieurs résultats pour en créer une réponse enrichie. La combinaison des différents textes permet de passer toutes les informations à la brique suivante.

![experience_internal_action](@public/experiences/merge.png)


### Saisir un prompt

Envoyez une consigne personnalisée à l’IA pour générer du contenu ou transformer des données.

Le prompt est utilisé : 

- sur l’action [“Poser une question à l’IA”] en tant qu’instruction
- sur l’action [Combiner les résultats] en tant qu’instruction

![experience_internal_action](@public/experiences/ai.png)

<Callout>
Sur l'action `Combiner des résultats`, vous pouvez combiner plusieurs résultats mais également ajouter un prompt d'instruction pour guider l'IA de la prochaine brique.
</Callout>

![experience_internal_action](@public/experiences/merge-tip.png)

Ici dans cet exemple le resultat de la brique `combiner des résultats` est un texte avec un prompt d'instruction suivi par les deux réponses de l'utilisateurs aux questions des briques `Saisir une entrée`.


### Poser une question à l’IA

Interrogez directement l’IA sans passer par un agent à configurer pour obtenir une réponse rapide ou une recommandation.

Sur ce bloc, vous verrez 2 points en entrée : 

- Le premier doit être relié à un prompt car il s’agit de l’instruction que vous devez donner à l’IA.
- Le deuxième doit être relié à la sortie du bloc que vous souhaitez mettre en contexte de la question

Choisissez ensuite le LLM que vous souhaitez interroger. Vous aurez le choix entre les LLM que votre organisation aura ajouté. 

![experience_internal_action](@public/experiences/ai.png)

### Détection d’intention

Analysez l’entrée de l’utilisateur pour identifier son besoin implicite et adapter le parcours automatiquement. 

![experience_internal_action](@public/experiences/intent.png)

Chaque intention aura son point de sortie. Elle devra donc être reliée à un parcours propre à lui. 

Pour chaque intention, renseignez : 

- l’intitulé de l’intention : c’est ce que l’IA viendra détecter dans le résultat de la brique précédente.
- la description de l’intention : pour donner plus de détails sur l’intention
